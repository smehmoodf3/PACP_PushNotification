ECLLogic.Pusher Class
Abstract definition of a component that pushes content to a BlackBerry via MDS. The Pusher class manages the HTTP connection, while the implementing subclass provides the content to push, names the port on the device to push to and sets the push headers. This pusher begins in a "construction" state where the beginGroup and addContact methods are used to build the message. Followin a call to finishedConstruction, this pusher transitions to a "sending" state where repeated calls to sendToHandheld push the content out to devices.

Access: Project
Base Classes: Object
  Members Description  
    log Define a static logger variable so that it references the Logger instance named "Pusher".

 
    DevicePort Provides the port on the handheld device that should receive pushes. Gets called once for each invocation of sendToHandheld, while in the "sending" state.

 
    Pusher Default constructor.

 
    beginGroup Starts a new group with the given name. Contacts added between the current and following invocations belong to that group. Can only be called while in the "construction" state.

 
    addContact Creates a new contact with the given data to the current group. Can only be called while in the "construction" state.

 
    finishedConstruction Transitions this pusher into its "sending" state. Can only be called while in the "construction" state.

 
    writeTo Writes the previously-constructed message to the given byte stream. Gets called once for each invocation of sendToHandheld. Can also be called from outside while in the "sending" state.

 
    establishRequestHeaders Adds any necissary headers to the provided http connection. That connection will subsequently be used for the push. Gets called once for each invocation of sendToHandheld, while in the "sending" state.

 
    sendToHandheld Pushes the already-constructed message to the indicated recipient. May be used many times, but only in the "sending" state.
