ECLLogic.BrowserChannelPusher Class
A pusher that formats its push message as an HTML page and selects a device port and http headers for a channel push of that page. It is derived from the base class Pusher

Access: Project
Base Classes: Pusher
  Members Description  
    log Define a static logger variable so that it references the Logger instance named "BrowserChannelPusher".

 
    PUSH_TYPES Array which holds the push type values Browser-Channel and Browser-Channel-Delete

 
    DevicePort Returns the port at which the device is listening

 
    BrowserChannelPusher Default constructor. Begins the HTML page.

 
    beginGroup Adds the group name to the HTML page.

 
    addContact Adds the contact details to the HTML page.

 
    finishedConstruction Ends the HTML page and stores the file if any location is specified.

 
    writeTo Writes the finished HTML page to the output stream.

 
    establishRequestHeaders Adds the headers during the push operation.

 
    storeFinishedMessage Saves the constructed webpage content to a file on the local machine. The file name is retrieved from the channel property StoreLocation. The storeFinishedMessage operation is safe in that it logs any raised exceptions but does not rethrow them. So if an invalid filename is specified, it won't kill the whole push operation.

 
    _workingMessage This stores message while it is under construction.

 
    _finishedMessage The stores the message after it is complete.
